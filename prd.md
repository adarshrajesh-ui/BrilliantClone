Expected Value Lab - MVP PRD Page 1 PRD Expected Value Lab: A learn-by-doing MVP for one introductory probability chapter MVP The MVP is a login-based web app that teaches one chapter: Expected Value - The Average Outcome of Uncertainty. It turns a textbook-style chapter into 8 visual, interactive problems that progressively move the learner from observing a simulation to independently building and interpreting an expected value model. The app must teach without AI. All problems, hints, answer checks, and feedback are hand-built and deterministic. What is in scope for MVP vs what is NOT In scope for MVP - - Google sign-in; user profile creation; saved progress per user. - - One chapter on expected value with 8 interactive, visual problems. - - Visual-first learning: spinner, mystery boxes, tables, formula builder, balance scale, fairness number line, risk comparison graph. - - Instant feedback under 100ms, hand-written hints, deterministic answer checking. - - Completion %, attempts, hints used, mistake type, mastery state, streak, milestones. - - Mobile-responsive design. All drag/drop interactions must also support tap-to-select and tap-to-place; on mobile, tap is default and drag/drop is optional polish. - - Deployed public web app using React + Firebase. Explicitly NOT in scope for MVP - - No AI tutor, AI hints, AI-generated problems, or model calls. - - No full probability course, multiple complete chapters, videos, long readings, payments, leaderboards, teacher dashboards, social features, mobile app store release, or advanced probability theory. - - No dependency on drag/drop for correctness; every interaction must have a tap fallback. User persona Primary learner: an introductory probability or statistics student who has seen EV formulas but does not understand what the numbers mean. They need visual setups that clarify outcomes, probabilities, payout, profit, and fairness. Secondary learner: a visual math learner who struggles when math is only presented as notation and needs to manipulate examples before the formula becomes intuitive. Domain Domain: introductory probability. Specific subject: expected value. Chapter goal: make EV intuitive as a long-run average and a weighted average for decision-making under uncertainty. Use stories: focused and not focused Focus stories: sign in and resume; start the EV chapter; interact with visual probability setups; get instant specific feedback; use visual hints; complete 8 problems; see progress, mastery, streak, and suggested review area. Won't focus stories: ask an AI tutor; browse many courses; create lessons; compete with friends; watch videos; manage classrooms; learn advanced probability topics. MVP only: one login-based, hand-built visual chapter on expected value. No AI in MVP. Expected Value Lab - MVP PRD Page 2 MVP Problem Specs Five PRD pages are dedicated to exact problem behavior and answer rules. Problems page 1 of 5: observe -> predict -> construct Problem 1 - The Long-Run Average Concept: EV is the average outcome over many repetitions. Scenario: Bob plays a spinner game with two equal sections: win $10 or win $0. Visual + interaction - - Spinner split 50/50; counters for spins, total winnings, average per spin; running average line graph. - - Buttons: Spin once, Spin 10 times, Spin 100 times. Each updates animation, total, average, and graph. - - Learner predicts the long-run average before running many spins. Answer and completion rules - - Completion rule: submit prediction, run at least 100 total spins, and identify $5 as the long-run average. - - Accepted answer format: multiple choice $0, $5, $10; normalized correct value = 5. - - Mistake types: chose extreme outcome; confused a single spin result with the long-run average. - - Feedback: explain that equal chance of $10 and $0 averages halfway between them. Problem 2 - Build the Weighted Average Concept: EV is a weighted average of outcomes. Scenario: spinner changes to 25% chance of $20 and 75% chance of $0. Visual + interaction - - Spinner with one quarter labeled $20 and three quarters labeled $0. - - Formula builder: EV = ___ x ___ + ___ x ___. - - Learner uses tap-to-select/tap-to-place cards: $20, $0, 25%, 75%. Drag/drop may also be supported but is not required. Answer and completion rules - - Completion rule: correctly place both outcome-probability pairs and submit final EV. - - Accepted answer format: formula arrangement plus numeric result; accepted values for EV: 5, 5.0, $5. - - Correct model: 20 x 0.25 + 0 x 0.75 = 5. - - Mistake types: reversed outcome/probability; omitted probability; used 20 as answer because it is the largest payout. MVP only: one login-based, hand-built visual chapter on expected value. No AI in MVP. Expected Value Lab - MVP PRD Page 3 MVP Problem Specs Problem behavior must be deterministic and agent-buildable. Problems page 2 of 5: convert visuals -> calculate Problem 3 - Mystery Box Outcomes Concept: EV can be calculated from counts, not just percentages. Scenario: Bob chooses one of 6 boxes: 1 has $12, 2 have $6, 3 have $0. Visual + interaction - - Six clickable boxes; learner taps to reveal prizes. - - App groups boxes by outcome and shows a table: Outcome | Number of Boxes | Probability. - - Learner fills count and probability for each row. Matching boxes highlight when each row is active. Answer and completion rules - - Completion rule: reveal all boxes and complete all table cells correctly. - - Accepted answer format: table entries; probabilities accepted as fractions or decimals: 1/6 or 0.1667, 2/6 or 1/3 or 0.3333, 3/6 or 1/2 or 0.5. - - Correct rows: $12 -> 1 box -> 1/6; $6 -> 2 boxes -> 2/6; $0 -> 3 boxes -> 3/6. - - Mistake types: entered counts as probabilities; probabilities do not sum to 1; ignored $0 outcome. Problem 4 - Calculate EV from the Table Concept: once probabilities are identified, calculate EV by summing each outcome x probability contribution. Scenario continues from Problem 3. Visual + interaction - - Boxes collapse into colored EV chunks: 12 x 1/6 + 6 x 2/6 + 0 x 3/6. - - Learner fills intermediate contribution cells, then final EV. - - Visual colors link each contribution to the matching outcome group. Answer and completion rules - - Completion rule: submit all three row contributions and the final EV. - - Accepted answer format: numeric cells. Contributions: 2, 2, 0. Final EV accepted: 4, 4.0, $4. - - Correct model: 12 x 1/6 = 2; 6 x 2/6 = 2; 0 x 3/6 = 0; EV = 4. - - Mistake types: arithmetic error; omitted $0 row; summed payouts without weighting by probability. MVP only: one login-based, hand-built visual chapter on expected value. No AI in MVP. Expected Value Lab - MVP PRD Page 4 MVP Problem Specs The middle problems add cost, profit, and classification. Problems page 3 of 5: payout -> profit -> decision Problem 5 - Expected Payout vs Expected Profit Concept: expected payout and expected profit are different when there is a cost to play. Scenario: the mystery box game has expected payout $4 and costs $3 to enter. Visual + interaction - - Balance scale or equation strip: expected payout +$4, cost -$3. - - Learner taps the cost block into the equation and enters expected profit. - - App visually shows cost pulling the result down from $4 to $1. Answer and completion rules - - Completion rule: select the formula payout - cost and submit expected profit. - - Accepted answer format: numeric: 1, 1.0, $1. - - Correct model: expected profit = expected payout - cost = 4 - 3 = 1. - - Mistake types: answered payout $4; added cost instead of subtracting; treated cost as probability. Problem 6 - Fair, Favorable, or Unfavorable? Concept: positive expected profit is favorable, zero is fair, negative is unfavorable. Scenario: three games have expected payout and cost already shown. Visual + interaction - - Three game cards: A payout $5 cost $5; B payout $7 cost $5; C payout $3 cost $5. - - Each card includes payout bar, cost bar, and hidden profit meter. - - Learner sorts cards into Favorable, Fair, Unfavorable using tap-to-select/tap-to-place buckets. Answer and completion rules - - Completion rule: place all three cards into correct buckets. - - Accepted answer format: classification: A=fair, B=favorable, C=unfavorable. - - Correct profit values: A = 0, B = +2, C = -2. - - Mistake types: treated any positive payout as favorable; confused fair with favorable; forgot to subtract cost. MVP only: one login-based, hand-built visual chapter on expected value. No AI in MVP. Expected Value Lab - MVP PRD Page 5 MVP Problem Specs The learner now builds the whole model with much less scaffolding. Problems page 4 of 5: full EV model Problem 7 - Build the Whole EV Model Concept: independently convert a game into an EV calculation. Scenario: a carnival wheel has 10 equal sections: 1 section pays $30, 2 sections pay $10, 7 sections pay $0. It costs $5 to spin. Visual + interaction - - A 10-section wheel is shown. Learner taps sections to group them by payout. - - Blank table: Outcome | Probability | Contribution. - - Below table: Expected payout, Cost, Expected profit, Decision. - - The formula is not pre-filled. Learner must infer probabilities from wheel sections and build row contributions. Answer and completion rules - - Completion rule: fill all probabilities, all contributions, expected payout, expected profit, and final decision. - - Accepted answer format: table + numeric fields + classification. - - Probabilities: $30 -> 1/10 or 0.1; $10 -> 2/10 or 0.2; $0 -> 7/10 or 0.7. - - Contributions: 3, 2, 0. Expected payout: 5. Expected profit: 0. Decision: fair. - - Mistake types: wrong denominator; counted sections but did not convert to probability; calculated payout but not profit; marked fair game as favorable because payout is positive. Visual hint rules - - Probability hint: highlight selected outcome sections and show selected sections / 10 total. - - Contribution hint: animate outcome x probability for the active row. - - Profit hint: show payout $5 and cost $5 balancing to zero. - - Decision hint: place expected profit dot at 0 on the fairness number line. Expected Value Lab - MVP PRD Page 6 MVP Problem Specs The final problem checks conceptual transfer beyond formula use. Problems page 5 of 5: compare same EV, different risk Problem 8 - Same Expected Value, Different Risk Concept: EV does not describe the full experience of uncertainty. Two games can have the same EV but different risk. Scenario: Game A gives guaranteed $5. Game B gives 50% chance of $10 and 50% chance of $0. Visual + interaction - - Two cards: A has a solid bar at $5; B has a split bar between $10 and $0. - - Learner runs 20 simulated trials for each game. A stays flat; B jumps between $0 and $10. - - Both running-average graphs approach $5, but Game B has visible spread. Answer and completion rules - - Completion rule: run both simulations, submit EV for each game, choose higher-risk game, and select the correct reason. - - Accepted answer format: numeric EV fields, multiple choice risk field, multiple choice explanation. - - Correct answers: EV(A)=5; EV(B)=5; more risk=Game B; reason=variable outcomes despite same long-run average. - - Mistake types: said Game B has higher EV because it can pay $10; said games are identical because EV is identical; confused average with guaranteed result. Cross-problem visual help and feedback rules All help must be visual-first. Probability mistakes highlight matching sections/boxes. Outcome-probability confusion highlights dollar values vs frequency. Payout-profit confusion uses a balance scale. Fairness mistakes use a negative/zero/positive number line. Risk mistakes compare a flat line to a jagged line with same average. Feedback must be specific and deterministic. Bad: "Wrong." Good: "You used the number of winning sections as the probability. Since the wheel has 10 total sections, divide by 10." MVP only: one login-based, hand-built visual chapter on expected value. No AI in MVP.
